# QA/QC Notes for Ocean Carbonate Chemistry Data

Credible ocean carbonate data require strong analytical metadata and quality control.

## Recommended fields

- sample identifier
- station identifier
- date and time
- latitude and longitude
- depth
- temperature
- salinity
- pressure
- pH value and pH scale
- pH method
- DIC
- total alkalinity
- pCO2 or fCO2
- oxygen
- nutrients
- calcium when measured or assumed
- instrument identifier
- calibration information
- certified reference material batch
- sample preservation
- analysis date
- uncertainty estimate
- quality-control flag
- data-processing version

## Common quality issues

- pH scale not specified
- mixing total, free, and seawater pH scales
- missing temperature or salinity context
- treating sensor data as drift-free
- using alkalinity without titration method details
- using DIC without preservation and handling metadata
- applying open-ocean constants to estuarine systems without caution
- ignoring nutrients and pressure in carbonate calculations
- failing to propagate uncertainty
- comparing values from incompatible methods

## Practical principle

Ocean carbonate data should allow another qualified reviewer to reconstruct how the sample was collected, preserved, analyzed, calibrated, corrected, and interpreted.
