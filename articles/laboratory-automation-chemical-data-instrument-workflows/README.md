# Laboratory Automation, Chemical Data, and Instrument Workflows

This article folder supports the Chemistry knowledge-series article **Laboratory Automation, Chemical Data, and Instrument Workflows**.

The workflows use synthetic educational data to demonstrate laboratory automation audits, instrument run manifests, sample queues, metadata completeness checks, QC drift analysis, instrument utilization, data provenance, and full-stack scientific-computing scaffolding.

## Repository structure

- `python/` — workflow audit, metadata completeness, QC review queues, and provenance manifests
- `r/` — instrument utilization, runtime summaries, and QC drift analysis
- `julia/` — throughput and completeness metrics
- `fortran/` — compact workflow reliability kernel
- `rust/` — automation manifest validator
- `go/` — CSV audit utility for run manifests
- `c/` — turnaround and completion metric utility
- `cpp/` — QC flag classifier
- `sql/` — schema and queries for samples, instruments, runs, files, QC checks, and audit trails
- `docs/` — methodology, reproducibility, and responsible-use notes
- `data/` — synthetic educational laboratory automation data
- `notebooks/` — notebook scaffolds and report notes
- `outputs/` — generated tables, reports, figures, and manifests

## Responsible-use note

All data in this folder are synthetic and educational. These workflows are not validated for clinical, environmental-compliance, forensic, pharmaceutical-release, industrial-quality, or safety-critical laboratory decisions.
