# QA/QC Notes for Soil Chemistry Data

Credible soil chemistry data require strong sampling design and method metadata.

## Recommended fields

- sample identifier
- site identifier
- latitude and longitude
- land use and management history
- crop or vegetation cover
- sampling date
- depth interval
- horizon designation when available
- sampling method
- composite or grab strategy
- bulk density
- texture
- analyte
- unit
- laboratory method
- extraction method
- detection limit
- reporting limit
- uncertainty
- quality-control flags
- laboratory identifier
- chain-of-custody status when relevant
- data-review status

## Common quality issues

- comparing different depths without harmonization
- comparing different phosphorus extraction methods
- ignoring bulk density in soil carbon stock estimates
- treating total metal concentration as bioavailable concentration
- omitting rock-fragment correction
- failing to distinguish agronomic and environmental interpretations
- using pH values without method context
- interpreting nitrate without sampling date and moisture context
- ignoring spatial heterogeneity
- using carbon-stock changes without uncertainty estimates

## Practical principle

Soil chemistry data should allow another qualified reviewer to reconstruct where the sample came from, what depth it represents, which method produced the value, what unit was used, and what interpretation is scientifically justified.
