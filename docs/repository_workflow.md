# Chemistry Repository Workflow

For each article:

1. Create or update the article folder under articles/article-slug/.
2. Add only useful code examples to the article body.
3. Put expanded computational workflows in the GitHub repository.
4. Keep SQL, metadata schemas, provenance tables, and larger workflows in the repository rather than the WordPress article body.
5. Use synthetic or public-domain data unless a source license clearly permits reuse.
6. Include clear comments, assumptions, limitations, and responsible-use notes.
7. Commit and push after each article update.

Suggested article-folder subdirectories:

- python/
- r/
- julia/
- fortran/
- rust/src/
- go/
- c/
- cpp/
- sql/
- notebooks/
- docs/
- data/
- outputs/tables/
- outputs/figures/
- outputs/reports/
- outputs/manifests/
